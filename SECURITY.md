# Security Policy

## Supported Versions
We actively patch security vulnerabilities in the following major version branches:

| Version | Supported |
| :--- | :--- |
| v1.x.x | Yes |
| < v1.0.0 | No |

---

## Reporting a Vulnerability
We take the security of our microservices architecture seriously. If you find a security vulnerability, please do **not** open a public GitHub issue. Instead, report it privately to the project maintainers.

Please include the following details in your report:
*   Domain / Microservice affected (e.g. `auth`, `payments`).
*   Description of the vulnerability (e.g. SQL Injection, JWT Signature Bypass, Cache Eviction vulnerability).
*   Steps to reproduce the vulnerability (proof of concept).
*   Potential impact.

We will acknowledge receipt of your vulnerability report within 48 hours and work with you to patch the issue before making it public.
